Conceitos Básicos de Redes

Redes de computadores conectam dispositivos permitindo comunicação e compartilhamento de recursos. Entender os conceitos básicos é fundamental para diagnóstico e suporte técnico eficiente.

  • IP (Internet Protocol): Endereço lógico que identifica um dispositivo numa rede.
  • MAC (Media Access Control): Identificador físico único da placa de rede.
  • DHCP (Dynamic Host Configuration Protocol): Protocolo que atribui IPs automaticamente.
  • DNS (Domain Name System): Sistema que converte nomes de domínio em endereços IP.
  • Gateway: Roteador ou dispositivo que conecta redes diferentes.
  • Subnet Mask: Máscara que define o tamanho da rede local.

Principais Protocolos de Rede

  • TCP (Transmission Control Protocol): Protocolo orientado à conexão que garante entrega confiável de dados.
  • UDP (User Datagram Protocol): Protocolo sem conexão, mais rápido, usado para streaming e jogos.
  • HTTP/HTTPS: Protocolos para transferência de páginas web.
  • FTP: Protocolo para transferência de arquivos.
  • SSH: Protocolo para acesso remoto seguro.
  • DHCP: Para atribuição automática de IPs.
  • DNS: Tradução de nomes para IPs.

Portas TCP e UDP - Guia Completo

Portas são números usados para identificar serviços e aplicações dentro de um dispositivo na rede, possibilitando a comunicação adequada entre dispositivos. Elas podem ser TCP (confiáveis) ou UDP (mais rápidas, sem confirmação).

As portas são numeradas de 0 a 65535, e são divididas em:

  • Portas bem conhecidas (Well Known Ports): 0 a 1023 - Usadas por serviços padrão (HTTP, FTP, SMTP, etc).
  • Portas registradas (Registered Ports): 1024 a 49151 - Usadas por aplicações específicas.
  • Portas dinâmicas/privadas (Dynamic/Private Ports): 49152 a 65535 - Usadas temporariamente por aplicações clientes.

Como as portas funcionam no TCP/IP

O modelo TCP/IP utiliza portas para multiplexar conexões em uma mesma máquina, permitindo que vários serviços funcionem simultaneamente usando o mesmo endereço IP. Cada pacote enviado ou recebido carrega uma porta de origem e uma porta de destino.

Por exemplo, quando você acessa um site via HTTPS, o seu navegador cria uma conexão de origem numa porta dinâmica (ex: 53215) e conecta à porta 443 (HTTPS) do servidor.

Avançado: Portas, Firewall e Segurança

É importante saber quais portas abrir no firewall para garantir que apenas os serviços necessários fiquem acessíveis. Portas abertas desnecessariamente aumentam o risco de ataques.

  • Scans de portas: Ferramentas como Nmap identificam portas abertas, usadas para auditoria.
  • Firewalls: Bloqueiam ou permitem tráfego baseado em portas e protocolos.
  • Port knocking: Técnica para abrir portas dinamicamente com sequência de pacotes.
  • Portas padrão: Podem ser alteradas para dificultar ataques.

Exemplos Práticos

  • Limpar cache DNS: ipconfig /flushdns
  • Testar conectividade e tempo de resposta: ping -n 5 google.com
  • Ver rota até host: tracert google.com
  • Exibir conexões ativas: netstat -ano
  • Consulta DNS detalhada: nslookup -type=mx gmail.com
  • Ver portas abertas e conexões: netstat -an | findstr LISTENING
  • Testar porta específica (ex: 80): telnet google.com 80

Scripts e Dicas para Redes

Automatize tarefas comuns de rede com scripts simples para diagnóstico e manutenção.

1. Script para renovar IP e limpar cache DNS (Windows)

@echo off
echo Liberando endereço IP atual...
ipconfig /release
echo Renovando endereço IP...
ipconfig /renew
echo Limpando cache DNS...
ipconfig /flushdns
echo Operação concluída.
pause

2. Script para monitorar conexões abertas

@echo off
echo Monitorando conexões de rede...
netstat -ano
pause

3. Script para verificar a rota até um host

@echo off
set /p host=Digite o endereço ou IP do host: 
tracert %host%
pause
